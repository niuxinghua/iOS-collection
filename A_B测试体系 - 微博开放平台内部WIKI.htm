Paladin系统介绍

Paladin系统集成A/B测试和人工评测，能灵活组织管理测试任务；方便快捷地提供测试报告；对业务提供统一的处理框架。

什么是A/B测试?

A/B 测试简单来讲是在两个或多个产品的优化方案中选出最优方案的方法。

例如,期望优化产品的某个特性——通常是期望某项数据提升时,需要对产品某处进行调整。 对此可以提出若干版本,但是现有的理论无法判定哪种方案最佳,此时可将这几个版本同时上线, 以小流量运行一段时间获得数据反馈,来判断版本优劣。

什么是人工评测?

人工评测是指人为地对产品或者策略进行评测分析、对比验证，从而帮助建立智能的模型。

为什么要进行A/B测试和人工评测?

A/B 测试可以使你在现有流量中获取更高的收益。每个产品都拥有自己的目标,也就是这个产品存在的意义。

A/B测试这种方法论的应用非常广泛，这里讨论在Web产品、移动产品、数字广告优化领域的应用。应用场景由小到大可以可以分为：

  • 元素/控件层面
  • 功能层面
  • 产品层面
  • 公司层面

A/B测试主要通过开关和指标进行策略或者方案择优，而人工评测更加量化和可视化，可以认为是A/B测试的衍生品。 人工评测更为客观地检测产品的质量好坏，并且还可获取样本集进行机器学习，建立业务模型。新的垂直业务或者迭代频繁的产品需要快速做出好的决策，这时人工评测是一种很好的工具。

术语

实验组

试验组用来组织和管理具有相同测试目标的一组实验。 A/B测试实验具有相同的模块开关和优化指标。人工评测实验具有相同的评测标签和任务类型。

实验

一个A/B测试实验代表了一个固定比例流量上具有某种模块开关组合的运行实例。 例如,测试购买按钮在红绿蓝三种颜色情况下的转化率,而当前购买按钮的颜色是蓝色,就需要建立两个实验,一个代表红色按钮,一个代表绿色按钮,这两个实验和默认测试数据进行对比分析，就可以得出红绿蓝三种颜色哪个更利于转化率提升的结论。

一个人工评测实验代表了一组评测数据的运行实例，这组数据分配多个评测人，每个评测人有独立的评测任务。 例如，现在需要统计话题实时流的垃圾率，选择数据来源（扫日志或调用接口）得到一批微博，随机均匀分配给评测人。评测人对每条微博标记好、中、差评，最终汇总统计结果，得到差评率。

模块开关

模块开关是根据实验的优化指标和目标对象确定的各种测试变量。一个模块开关可以用于多个实验之中。 在一个典型的简单A/B测试活动中,多个相关的实验都只指定一个测试变量(开关),具有不同的变量值,代表所测试目标对象的不同方案。 如测试蓝微出的条数,可以定义一个 int 型的开关,当用户可以进入这个测试版本后,获取开关 的值,也许值是 20,此时出的微博条数为 20 条;当值为 10 时,出微博 10 条。

评测标签

评测标签是评测实验的一个变量，用来标志和分类评测内容。一个评测标签可以用于多个评测实验之中。 目前支持一级标签和二级标签的增删查改。

优化指标

指标是对业务数据的量化衡量计数。 如一个以应用内销售收入为核心业务指标的 App,它的关键指标可能包括:用户活跃度。 又如微博的一个指标:评论数。

Paladin整体架构

前提条件

  • 一个用户限定在一个系统内,如微博体系内。
  • 一个用户在一个时间能参与多个试验。
  • 一个用户,如果初次进入了某一实验的某个版本,二次启动后依旧会进入该版本。

使用步骤

Java SDK

初次使用时需要引入ab-test-service包，pom依赖如下： 

   <dependency>
       <groupId>com.weibo.api.ab-test</groupId>
       <artifactId>ab-test-service</artifactId>
       <version>RELEASE</version>
   </dependency>

XML

然后需要在spring的配置文件配置abtest rpc接口的XML文件： weibo-ab-test-rpc-service-client.xml。

内容如下：

引入client

接下来只需要引入ABTestClient即可使用相关功能 

   @Resource
   private ABTestClient abTestClient;

获取开关

  • 获取布尔型开关值 
   abTestClient.getBooleanSwitch(long uid, String switchName);
  • 获取整型开关值 
   abTestClient.getIntSwitch(long uid, String switchName);
  • 获取字符串型开关值 
   abTestClient.getStringSwitch(long uid, String switchName);

记录指标

  • 记录指标 
   abTestClient.recordMetric(long uid, String metricName, int count);
   abTestClient.recordMultiMetrics (long uid, Map<String, Integer> metrics)
  • 记录指标（不使用开关时） 
   abTestClient.recordMetricNoSwitch(long uid, String metricName, int count);
   abTestClient.recordMultiMetricNoSwitch (long uid, Map<String, Integer> metrics)
